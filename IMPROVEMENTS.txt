OVERVIEW
--------
This file documents every improvement made to the AURA AI Smart Complaint
Intelligence System. Changes were made across both the backend (Node/Express)
and frontend (React/Vite) to fix bugs, add real data, and polish the UI for
the hackathon demo.

--------------------------------------------------------------------------------
BACKEND CHANGES
--------------------------------------------------------------------------------

1. CRITICAL BUG FIX — complaintController.js (line 113)
   ---------------------------------------------------
   BEFORE:  constByCategory = await Complaint.aggregate([...])
   AFTER:   const byCategory = await Complaint.aggregate([...])

   Impact: The app was CRASHING every time anyone visited the Admin Dashboard
   because `constByCategory` is not valid JavaScript (missing space between
   `const` and the variable name). This is now fixed — the /api/complaints/stats
   endpoint works correctly.

2. NEW ENDPOINT — GET /api/complaints/trends
   -----------------------------------------
   File: backend/controllers/complaintController.js  (new getTrends function)
   File: backend/routes/complaintRoutes.js  (new route added)

   What it does: Returns an array of 7 objects, one for each of the last 7 days,
   with the count of complaints submitted that day. Uses MongoDB's $dayOfWeek
   aggregation and filters by createdAt >= 7 days ago.

   Example response:
   [
     { "name": "Thu", "complaints": 2 },
     { "name": "Fri", "complaints": 5 },
     ...
   ]

   Access: Protected, Admin only (same as /stats).

FRONTEND CHANGES
--------------------------------------------------------------------------------

3. api.js — Two new API functions added
   -------------------------------------
   File: frontend/src/services/api.js

   a) deleteComplaint(id, token)
      Calls DELETE /api/complaints/:id — the backend route existed but was never
      wired into the frontend. Admins can now delete complaints from the UI.

   b) fetchTrends(token)
      Calls GET /api/complaints/trends to get the 7-day real data for the
      Admin Dashboard area chart.

4. AdminDashboard.jsx — Real data + new stat card + polish
   --------------------------------------------------------
   File: frontend/src/pages/AdminDashboard.jsx

   Changes:
   - Area chart now uses LIVE data from /api/complaints/trends instead of the
     hardcoded Mon-Sun dummy array (40, 30, 45, 28, 55, 20, 15).
   - Chart title updated to "7-Day Complaint Volume" with subtitle "Live data
     from the last 7 days".
   - Loading skeleton added for chart while data is fetching.
   - "Avg. Response Time" stat card replaced with "Resolution Rate" — a real
     metric calculated as (resolved / total) * 100%. Shows a live percentage.
   - Priority badges added to the recent complaints table (was missing priority
     column before). Color-coded: High=red, Medium=orange, Low=blue.
   - Loading skeletons added to the table rows while data loads.
   - "View Intelligence Center" button changed from a plain button to a proper
     anchor link pointing to /complaints.

5. App.jsx — Analytics page rebuilt, Alerts page rebuilt
   ------------------------------------------------------
   File: frontend/src/App.jsx

   a) Analytics Page (was: placeholder text)
      Now shows:
      - Three summary stat cards: Resolution Rate (with progress bar), Resolved
        count, Pending count. All populated from /api/complaints/stats.
      - "Complaints by Category" horizontal bar chart using real byCategory data
        from the stats endpoint. Each bar is color-coded.
      - "Sentiment Distribution" donut pie chart built from all complaints data,
        grouped client-side by sentiment. Color mapping: Positive=green,
        Neutral=grey, Negative=orange, Urgent=red.

   b) Alerts Page (was: two hardcoded dummy alert boxes)
      Now shows:
      - Two system status banners (operational notice + maintenance notice).
      - "Active High-Priority Complaints" live feed — fetches all complaints,
        filters to those where priority='High' OR sentiment='Urgent' AND status
        is still 'pending'. Displayed as a live list with red pulse indicators.
      - "X open" badge showing real count.
      - Empty state with green checkmark if no urgent issues exist.
      - Loading skeletons while data is fetching.

6. Complaints.jsx — Delete button, dark mode, expandable AI response
   ------------------------------------------------------------------
   File: frontend/src/pages/Complaints.jsx

   Changes:
   a) Delete button for admins
      Each complaint row now has a trash icon button (Trash2) for admins.
      Clicking it shows a browser confirm dialog, then calls DELETE /api/complaints/:id.
      The row disappears immediately from the list on success (optimistic update).
      The button is disabled while deletion is in progress.

   b) Dark mode support
      The entire table and search/filter bar now properly support dark mode.
      All hardcoded `bg-white` and `text-gray-` classes now have `dark:` variants
      (dark:bg-slate-800, dark:text-gray-200, dark:border-slate-700, etc.).

   c) Expandable AI Response panel
      A chevron button (down/up) is added per row. Clicking it expands an inline
      row below showing the AI-generated suggested response for that complaint,
      with a Bot icon. Only one row can be expanded at a time.

   d) Priority badge dark mode fix
      Priority badges (High/Medium/Low) now have proper dark: variants so they
      are legible in dark mode.

   e) Live stats in sidebar widget
      The "Automated Optimization" card now shows real resolved/pending counts
      from the loaded complaints array, instead of static text.

7. ComplaintForm.jsx — Character counter, priority colors, dark mode
   ------------------------------------------------------------------
   File: frontend/src/components/ComplaintForm.jsx

   Changes:
   a) Character counter
      Live counter shows "X/500 characters" below the textarea. Text turns
      orange when above 85% of the limit, red when at the limit. Input is
      capped at 500 characters (via .slice(0, MAX_CHARS)).

   b) Fixed priority badge colors
      Previously both Medium and Low priorities showed as blue. Now:
        High   → red (bg-red-50 / text-red-600)
        Medium → orange (bg-orange-50 / text-orange-600)   ← FIXED
        Low    → blue (bg-blue-50 / text-blue-600)

   c) Full dark mode support
      Textarea, result cards, error message box, and header all now have dark:
      variants for proper dark mode appearance.

   d) Voice input error feedback
      If voice recognition is not supported or fails, an error message is now
      shown to the user in the error box (previously only logged to console).

   e) Voice input guard
      Added a check before calling SpeechRecognition: if not supported, shows
      a user-visible error instead of crashing with "SpeechRecognition is not
      defined".
